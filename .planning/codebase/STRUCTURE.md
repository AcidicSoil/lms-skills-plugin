# Repository Structure

## Root Layout

- `src/` contains all authored plugin implementation code.
- `dist/` contains generated JavaScript, declaration files, and source maps.
- `.lmstudio/` contains LM Studio development bootstrap files.
- `.codex/` contains local GSD/Codex workflow configuration and agents.
- `.serena/` contains Serena project configuration.
- `.serena-gsd/` contains repository GSD support scripts.
- `docs/` contains OpenGSD agent documentation.
- `package.json`, `package-lock.json`, and `tsconfig.json` define the Node/TypeScript project.
- `manifest.json` defines LM Studio plugin metadata.
- `README.md` documents usage and exposed tools.
- `todo.md` contains the current Host/WSL workspace design proposal.

## Source Modules

- `src/index.ts`: composition root and plugin registration.
- `src/config.ts`: LM Studio configuration schema.
- `src/constants.ts`: shared paths, limits, defaults, regexes, and search weights.
- `src/types.ts`: domain and configuration interfaces.
- `src/pluginTypes.ts`: local LM Studio controller/context interfaces.
- `src/settings.ts`: persisted settings and effective configuration resolution.
- `src/setup.ts`: default skill-directory bootstrap.
- `src/scanner.ts`: skill discovery, metadata parsing, search, file reads, and directory traversal.
- `src/preprocessor.ts`: message extraction and prompt injection.
- `src/toolsProvider.ts`: all model-visible LM Studio tools.
- `src/executor.ts`: cross-platform shell resolution and process execution.

## Generated Output

`dist/` mirrors most modules from `src/` with four artifacts per module:

- `.js`
- `.js.map`
- `.d.ts`
- `.d.ts.map`

Generated output should not be treated as the source of truth; edit `src/` and rebuild.

## Naming Conventions

- Source filenames use lower camel case for multiword modules, such as `toolsProvider.ts` and `pluginTypes.ts`.
- Exported interfaces use PascalCase, such as `SkillInfo` and `EffectiveConfig`.
- Functions and local values use camelCase.
- Constants use uppercase snake case in `src/constants.ts`.
- LM Studio tool names use snake case, such as `read_skill_file` and `run_command`.

## Module Dependency Direction

- `src/index.ts` depends on configuration, tools, preprocessing, setup, constants, and plugin types.
- `src/toolsProvider.ts` depends on settings, scanner, executor, constants, and types.
- `src/preprocessor.ts` depends on settings, scanner, constants, and types.
- `src/settings.ts` depends on config, constants, and types.
- `src/scanner.ts` depends on Node filesystem/path APIs, constants, and types.
- `src/executor.ts` depends on Node process/filesystem APIs and constants.
